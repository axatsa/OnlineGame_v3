## Сессия 26 апреля 2026

### Роль org_admin — B2B панель для администраторов организаций

**Новая роль в системе.** Добавлена третья роль `org_admin` — для представителя школы или организации, купившей тариф SCHOOL ($49/мес). Теперь в системе три роли: `super_admin`, `org_admin`, `teacher`.

**Backend — новый роутер `/org-admin`.**
- `GET /org-admin/me` — статистика организации: занято/всего мест, учителей, токенов за месяц, срок действия
- `GET /org-admin/teachers` — список только своих учителей
- `POST /org-admin/teachers` — создать учителя (проверка лимита `license_seats`, инкремент `used_seats`)
- `POST /org-admin/teachers/{id}/toggle-block` — блокировка/разблокировка
- `DELETE /org-admin/teachers/{id}` — удаление + декремент `used_seats`
- `POST /org-admin/invite` — генерация инвайт-ссылки для своей организации
- `GET /org-admin/contact` — возвращает `admin_telegram` из GlobalSetting

**Backend — новые эндпоинты super_admin.**
- `POST /admin/teachers/{id}/promote` — назначить учителя org_admin его организации
- `POST /admin/teachers/{id}/demote` — вернуть роль teacher
- `POST /admin/organizations/{org_id}/set-token-limit` — установить лимит токенов всем учителям организации сразу

**Frontend — новая страница `/org-admin`** (`OrgAdminDashboard.tsx`):
- Шапка с названием орги, кнопками "Купить план", "Написать в Telegram" и "Выйти"
- 3 стат-карточки: занято мест, кол-во учителей, токенов за месяц
- Баннер "Нужно больше токенов?" с кнопкой написать администратору
- Прогресс-бар использования лицензий (со сроком истечения)
- Таблица учителей: имя/email, школа, лимит токенов, статус, блок/удалить
- Модалка добавления учителя (email, имя, пароль, школа)
- Модалка инвайт-ссылки с кнопкой копировать

**Frontend — модалка "Купить план"** в OrgAdminDashboard:
- Вкладка PRO (на учителя): счётчик мест ±, быстрые пресеты [1,3,5,10], итого = seats × $15, подсказка "SCHOOL выгоднее" когда $49 < total
- Вкладка SCHOOL (вся орга): список фич, $49/мес, кнопка перейти к оплате
- Оба варианта ведут на `/checkout?plan={pro|school}`

**Frontend — AdminPanel улучшения:**
- Бейдж "ORG ADMIN" в строке учителя если `role === "org_admin"`
- Кнопки "Сделать адм.орги" / "Снять роль" (только если у учителя есть `organization_id`)
- Модалка установки токен-лимита организации: пресеты 30k/50k/100k/200k
- Настройка `admin_telegram` в системных настройках
- Toast с подсказкой "Пользователь должен перезайти" после promote/demote
- Исправлен редирект при impersonation: учитывает роль (`/admin`, `/org-admin`, `/teacher`)

### MathPuzzle — таймер, серия, экран результатов

**Таймер обратного отсчёта** (20 секунд на вопрос) с анимированной полосой прогресса и сменой цвета (зелёный → жёлтый → красный). Если время вышло — задание автоматически засчитывается как неверное.

**Серия (streak) с множителем очков.** Каждый правильный ответ подряд увеличивает серию. Множитель ×1/×2/×3 применяется к итоговым очкам. Иконка Flame и подсказка появляются когда серия ≥ 2. Неверный ответ сбрасывает серию в 0.

**Экран результатов** после завершения всех раундов:
- Оценка (🏆/🌟/👍/💪) в зависимости от точности
- Общий счёт большим шрифтом
- Мини-статистика: правильных/всего, точность %, лучшая серия
- Таблица по каждому раунду: иконка верно/неверно, множитель серии, время, очки
- Кнопки "Назад" и "Ещё раз"
- Автоматический вызов `POST /gamification/activities/complete` на завершение

### AnalyticsPage — страница аналитики для учителей

Новая страница `/analytics` доступна учителям и org_admin.

- **4 стат-карточки:** всего генераций, за этот месяц, дней подряд (streak), использование токенов %
- **Прогресс-бар токенов** с цветом по уровню заполненности (зелёный/жёлтый/красный)
- **Граф активности за 14 дней** (recharts BarChart, автозаполнение дней без активности)
- **Круговая диаграмма** используемых типов игр
- **Топ-8 тем** из истории с прогресс-барами
- **Избранные генерации** (filtered by `is_favorite`)
- **Пустое состояние** с CTA если генераций ещё нет
- Кнопка "Аналитика" добавлена в навигацию TeacherDashboard

### Исправления пробелов в логике (gap audit)

**AuthContext обновляет данные при старте.** `initAuth` теперь вызывает `GET /auth/me` после чтения из localStorage — роль, лимиты токенов и другие поля обновляются при каждом открытии приложения без необходимости перелогиниваться.

**AnalyticsPage — живые данные токенов.** Страница теперь получает `tokens_used_this_month` и `tokens_limit` из `GET /payments/subscription/me` вместо устаревшего снимка из localStorage.

**OrgAdminDashboard — кнопка настроек.** Добавлена кнопка "Настройки" (иконка UserCog) в шапке → переход на `/profile`, где org_admin может сменить пароль и данные аккаунта.

**Маршруты `/profile`, `/history`, `/analytics`** теперь доступны для роли `org_admin` (ранее только `teacher`).

## Сессия 22 апреля 2026

### Регистрация и авторизация

**Бесплатный тариф без оплаты.** Кнопка "Начать бесплатно" на лендинге теперь ведёт на страницу регистрации, а не на оплату. Пользователь вводит имя, email и пароль — аккаунт создаётся сразу без платёжных данных.

**Страница восстановления пароля.** Добавлены две новые страницы — форма запроса сброса и форма ввода нового пароля. На странице входа появилась ссылка "Забыли пароль?", а на странице регистрации по инвайту — ссылка "Зарегистрироваться бесплатно".

**Бэкенд сброса пароля.** Созданы два новых эндпоинта: запрос сброса генерирует токен на 1 час и отправляет письмо, подтверждение сброса проверяет токен и обновляет пароль. Реализована защита от перебора email-адресов — ответ всегда одинаков независимо от существования аккаунта.

**SMTP для отправки писем.** Сервис отправки писем теперь поддерживает настоящий SMTP через переменные окружения. Если SMTP не настроен, ссылка для сброса выводится в консоль как заглушка.

**Rate limiting на авторизацию.** Эндпоинт входа ограничен 5 попытками в минуту с одного IP, эндпоинт запроса сброса пароля — 3 запросами в час.

**Исправлен редирект после регистрации по инвайту.** После успешной регистрации через ссылку-приглашение пользователь теперь попадает на дашборд учителя, а не на несуществующий маршрут.

### UX и навигация

**QR-код без авторизации.** Публичная страница для просмотра сгенерированных материалов — учитель показывает QR-код, ученик видит задание без входа в аккаунт.

**Мобильный вид генератора.** Нижняя панель с вкладками на мобильных, автопереключение на результат.

**Логотип, спиннер загрузки, таймаут на странице успешной оплаты** — UX-мелочи завершены.
