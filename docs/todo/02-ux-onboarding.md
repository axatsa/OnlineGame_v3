**Приоритет:** 🟡 Средний (Sprint 2)  
**Оценка:** ~4–6 дней  
**Исполнитель:** Frontend + Backend  
**Статус:** ❌ Не начато — нет ни флага в БД, ни UI компонента

## Контекст

Новый пользователь после входа не понимает с чего начать. Нужен приветственный онбординг и дашборд с реальной статистикой вместо заглушек.

---

## Подзадачи

### 2.1 Онбординг при первом входе

**Файлы:** `front/src/components/Onboarding/`, `front/src/context/AuthContext.tsx`

**Что делать:**
- Создать модальное окно `OnboardingModal.tsx` — 4 шага:
  1. **Добро пожаловать** — кратко о ClassPlay
  2. **Создай класс** — прямая кнопка для перехода к созданию класса
  3. **Попробуй генератор** — кнопка запустить первый AI-генератор
  4. **Запусти игру** — кнопка перейти к играм
- В БД добавить флаг `onboarding_completed: bool = False` в таблицу users
- API: `PATCH /api/v1/users/me/onboarding-complete` → устанавливает флаг
- Показывать модал если `user.onboarding_completed === false`
- Кнопка «Пропустить» тоже вызывает этот API

**Дизайн:**
- Прогресс-бар вверху (шаг 1 из 4)
- Иллюстрация или иконка на каждом шаге
- CTA-кнопки «Далее» / «Назад»

**Проверка:** Новый пользователь → онбординг показан. После завершения/пропуска — больше не показывается.

### 2.2 Dashboard с реальной статистикой

**Файлы:** `front/src/pages/dashboard/TeacherDashboard.tsx`, `backend/apps/generator/router.py`

**Что делать:**

**Backend — новый роут:**
```
GET /api/v1/stats/me
```
Ответ:
```json
{
  "total_generations": 47,
  "generations_this_month": 12,
  "games_launched": 8,
  "books_created": 3,
  "activity_by_day": [
    {"date": "2026-03-24", "count": 3},
    ...
  ],
  "top_features": [
    {"name": "Quiz", "count": 15},
    {"name": "Math", "count": 10}
  ]
}
```

**Frontend:**
- Карточки-метрики (всего генераций, в этом месяце, игр запущено)
- График активности по дням — простой bar chart (можно использовать `recharts`)
- Топ-3 используемых функции

**Примечание:** Таблица `generation_logs` уже существует в `apps/generator/models.py`. Данные уже пишутся.
